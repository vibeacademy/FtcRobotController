## 5. Insights and Learnings

- **Technical**: This project's `.mcp.json` already configures `memory` and `sequential-thinking` MCP servers — no setup was needed for `/log-session`'s memory integration. The memory store is shared across all local projects (311 entities at session end), so entity names should be prefixed or specific enough to avoid cross-project collisions.
- **Technical**: `/log-session`'s "After Writing" step references `/validate-memory`; porting one without the other leaves a dead reference. Command files can have inter-command dependencies worth checking before porting.
- **Process**: The entire `.claude/` directory (agents, commands, settings) is untracked in this repo. PR #20 starts tracking `commands/`; the rest (6 agent definitions, settings) still exists only on this machine. Fresh clones and teammates won't get the workflow tooling until it's committed.
- **Process**: Background-session pattern established for config changes: worktree → commit → push → draft PR for the git record, plus direct `cp` into the main checkout for immediate usability.
